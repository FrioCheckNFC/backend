2. En la entidad User (TypeORM):
   - Cambiar la columna:
     ```typescript
     @Column({
       name: 'role',
       type: 'text',
       array: true,
       default: '{}',
     })
     role: string[];
     ```

3. En los DTOs (por ejemplo, user.dto.ts, add-role.dto.ts, create-user.dto.ts, update-user.dto.ts):
   - Usar validadores para arrays:
     ```typescript
     @IsArray()
     @IsEnum(UserRole, { each: true })
     role?: UserRole[];
     ```
   - Si el DTO es para agregar varios roles, usar array. Si es para agregar uno, puede ser string.

4. En la lógica de negocio y servicios:
   - Donde antes usabas `user.role === 'VENDOR'`, ahora usa `user.role.includes('VENDOR')`.
   - Métodos para agregar/eliminar roles deben trabajar con arrays.

5. Migración de base de datos:
   - Crear una migración que cambie la columna `role` de string/enum a `text[]` (arreglo de texto).
   - Ejemplo de migración:
     - Renombrar columna original a `role_old`.
     - Crear nueva columna `role` como `text[]`.
     - Migrar datos: `UPDATE users SET role = ARRAY[role_old] WHERE role_old IS NOT NULL`.
     - Eliminar columna vieja.
     - (Si hay vistas dependientes, eliminarlas antes y recrearlas después).

6. Adaptar la vista `users_view` (si existe):
   - Ahora debe usar el campo `role` como array. Ejemplo:
     ```sql
     CREATE VIEW users_view AS
     SELECT *, role[1] AS main_role FROM users;
     ```

7. Probar todo el flujo:
   - Crear usuario con varios roles.
   - Consultar usuario y verificar que `role` es un array.
   - Probar endpoints que asignan, eliminan o consultan roles.
